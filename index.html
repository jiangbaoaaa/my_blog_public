Hybrid A*（混合 A*）

为什么要有 Hybrid A*

普通 A* 的路径长这样：在栅格中心之间走折线、走 45°。一辆真实的车（阿克曼转向，有最小转弯半径）根本走不出来。如果直接拿 A* 路径给控制器，要么跟踪不上，要么得做大量后处理（平滑、拟合），还可能拟合失败。

Hybrid A* 的目标：直接在车辆的”位姿空间”里搜索，让搜出来的路径本身就能被车开出来。

核心改动：状态从离散栅格 → 连续位姿

A* Hybrid A*
状态 栅格 (i, j) 连续位姿 (x, y, θ)，θ 是车头朝向
邻居生成 上下左右/对角 用车辆运动学前进一步（运动基元 motion primitive）
路径形状 栅格折线 真实可行驶轨迹（圆弧/RS 曲线段拼接）

每次扩展就是：从当前位姿，分别走这 6 段，得到 6 个候选新位姿。

连续空间怎么搜索？——3D 栅格去重

纯连续位姿有无限多个，没法搜。Hybrid A* 的巧思：

  • 把连续位姿 (x, y, θ) 投影到一个 3D 离散栅格（x 分箱、y 分箱、θ 分箱）。
  • 每个 3D cell 里只保留一个状态（通常是代价最低的那个，或第一个到达的）。
  • 这样搜索空间是有限的（分辨率完备），但每个状态本身又是连续的真实位姿——路径连续平滑，但搜索规模可控。

一句话：用离散栅格做”去重/记忆”，用连续状态做”路径生成”。这就是 “Hybrid（混合）” 二字的含义——离散 + 连续的混合。

启发函数要升级

普通欧氏距离在 Hybrid A* 里太弱（因为还要考虑朝向和不能横移），扩展会爆炸。常用更强的启发：

  1. 2D A* / Dijkstra 距离：先在 2D（忽略 θ、允许任意方向移动）上跑一遍，拿到每个格子到目标的距离，作为 h。这已经比欧氏强很多。
  2. Reeds-Shepp 距离：直接算”如果没障碍，从当前位姿到目标的最短 RS 曲线长度”。更强，但要配合碰撞检查裁剪。

工程上常把两者结合，甚至取多种启发的最大值，兼顾引导力和可采纳性。

Reeds-Shepp 加速：这是 “A* RS” 的 RS 部分

光靠运动基元一格一格扩，最后可能永远”差一点点”对不上目标的精确朝向 θ。解决办法：

  • 每次扩展出一个节点后，尝试用 Reeds-Shepp 曲线直接连到目标。
  • 如果这条 RS 曲线无碰撞，那就直接接上、结束搜索。
  • 这让算法能精确收敛到目标位姿，大幅减少靠近目标时的无谓扩展。

Reeds-Shepp 曲线：给定两个位姿，在”车能前进、能后退”的前提下，存在解析的最短路径（由若干段圆弧+直线组合，公式可查表）。能前进能后退 = 适合泊车这种需要揉库的场景。
Dubins 曲线：只能前进不能后退的对应物，适合单向行驶。

Hybrid A* 在规划框架中的定位与功能

经典 A* 在二维栅格 (x, y) 上搜索，相邻节点是上下左右四个格子。但这种路径对汽车没有意义——汽车受最小转弯半径约束，不能原地横移、不能任意转向。Hybrid A* 的解决办法是把朝向 θ 也纳入状态，搜索空间变成 (x, y, θ)，节点之间的转移边直接用满足运动学的基元（直行、定半径圆弧）。

在本模块里，Hybrid A* 是 makePlan 真正的内核。Day1 的节点层只负责把 costmap 和起终点喂进来、把输出包成 Trajectory；真正”在地图上找一条能走通、还能倒车的路径”是这里做的。Day2 的 RS 曲线在这里只担任启发函数——估计当前节点到终点还差多远，引导搜索朝终点走，避免盲目扩散。

关键类：AstarSearch 与 AstarNode

Hybrid A* 的核心数据结构有两层：一个是节点 AstarNode（搜索空间里的一个状态），一个是搜索器 AstarSearch（持有网格、待扩展队列、运动基元表）。

主循环的逻辑用一句话讲：反复取”目前看起来最有希望（cost 最小）”的节点，用运动基元往外扩一格，撞墙的扔掉、新的算上代价丢回队列，直到某个节点落进终点范围。 其中 cost = 已经走了多远（gc）+ 估计还剩多远（hc，RS 给的）。

calcReedsSheppDistance（L31）就是 Day2 那套：起一个 ReedsSheppStateSpace(radius)，调它的 distance() 拿到两点间 RS 最短距离。RS 距离之所以是好启发，是因为它本身就是满足运动学的最短路径长度（忽略障碍），比直线距离更贴近车的真实可走距离，能把搜索注意力有效拉向终点。

一个走读发现，写报告时注意区分：本实现里 RS 只当启发函数用（这一处），search 里没有“用 RS 曲线直连终点”的 analytical expansion（shoot）步骤——终点判定靠下面的 isGoal 范围盒。这和教科书 Hybrid A*（带 RS shoot）略有不同，是这份代码的简化。

Hybrid A* 搜索逻辑

状态空间：(x, y, θ) 三维网格离散

连续的位姿 (x, y, θ) 没法直接搜索，所以 setMap（L138）按 costmap 分辨率把 x, y 切成网格，再按 theta_size（默认 48）把航向 θ 切成 48 档，得到一个 3D 数组 nodes_[y][x][theta]。每个格子放一个 AstarNode。任意连续位姿都被 pose2index 吸附到最近的格子——这就是 Hybrid A* 控制状态数量、避免无限细分的关键。

代价的 gc 是连续累积的（不量化），但节点身份是按格子去重的：同一个格子只算一个节点（status 三态），所以不会无限扩展。

前进右弧 forward_left.flipped()（L102） 镜像得右转
倒车各向 .reversed()（L116） 前进各基元取反，得倒车版本

半径从 R_min 到 R_max 分 turning_radius_size+1 档（默认 12 档），每档有左、右两条弧，再加一条直行。每个 θ 档把这些基元用 rotated(theta) 转到当前朝向下（L107–112）；use_back=true 时再追加 reversed() 的倒车基元（L114–118）。

所有基元的弧半径都 ≥ R_min，所以每条转移边天然满足最小转弯半径约束——这是 Hybrid A* 路径”车能跟”的根本保证。

代价与启发

  • gc：从起点到当前节点的累积代价，每步加 move_cost。
  • move_cost：正常就是基元的 distance；如果这一步换了方向（is_back 翻转，即前进↔倒车切换），乘 reverse_weight（默认 2.0，L265）——这就是换向惩罚，让算法优先少换挡。
  • hc：estimateCost 给的 RS 距离 × distance_heuristic_weight。
  • cost = gc + hc，优先队列每次取最小。

reverse_weight 的存在，解释了为什么最终轨迹的”倒车段”是分段、不频繁的：换一次方向要付 2 倍代价，算法会尽量凑成”前进一长段 + 倒车一长段”，而不是反复进退。

终点判定与回溯

isGoal（L336）不是”恰好等于终点”，而是检查节点是否落进终点周围的范围盒：横向 < lateral_goal_range、纵向 < longitudinal_goal_range、航向差 < angle_goal_range（都除以 2 作半宽）。落进盒子就算到达。only_behind_solutions 还能要求解必须从终点后方进入。

命中后 setPath（L300）从终点节点沿 parent 指针一路回溯到起点，收集每个节点的位姿和 is_back，最后 std::reverse（L328）反转成起点→终点顺序。这里的 is_back 字段会一路传到 Day1 的节点层：createTrajectory 根据它给速度取正负号，getReversingIndices 再标出倒车切换点——倒车段的”源头”就在这一步。

RS 曲线在规划框架中的定位与功能

汽车不能像机器人那样原地旋转，必须遵守非完整运动学约束，即受限于最小转弯半径。

  • RS功能：在最小转弯半径约束下、允许前进 + 倒车，连接两个位姿 (x, y, θ) 的最短路径**。是 Dubins 曲线推广。
  • 为什么泊车要用它：侧方/倒车入库必须倒车，普通几何曲线（只前进）做不到；RS 给出”前进段 + 倒车段”拼接的最短解。
  • 在这个模块的作用：RS 是 freespace_planning_algorithms 算法库里的几何工具，RS曲线并非直接求解从起点到终点的最终路径，而是作为图搜索算法（A* RRT*）的节点拓展工具。主要负责两个功能：
    1. 启发函数（heuristic）：估”从当前点到终点还多远”，引导 A* 往终点搜（astar_search.cpp:227）。
    2. shoot / 终结：搜到某点时用 RS 直奔终点，命中即接上**（Day3 详读）**。

在 Hybrid A* 中，RS 曲线常被用作 Shot Heuristic (解析解剪枝)：当算法探索到一个靠近终点的节点时，如果不考虑障碍物，直接用 RS 曲线连向终点。如果这条 RS 曲线在 Costmap 上不碰撞，搜索就直接宣告成功，极大地加速了计算。

obstacle_cruise_planner：前方障碍 → 停车 或 减速跟车

定位与两层规划

行驶中，轨迹前方有障碍时由它处理。它把障碍分成两类，分别走两条规划路径：

  • 要停的：障碍挡死，必须在障碍前插一个零速点 → generateStopTrajectory
  • 要跟的：障碍在动（前车），算一个安全跟车限速，减速跟着 → generateCruiseTrajectory

PlannerInterface 是基类（planner_interface.hpp L41），巡航部分是纯虚函数 generateCruiseTrajectory（L72），有两个实现：OptimizationBasedPlanner（node.cpp L351，默认，用优化解速度）和 PIDBasedPlanner（L354，用 PID）。停车逻辑 generateStopTrajectory 写在基类里（L69），两种 planner 共用。

整条触发链在 onTrajectory（node.cpp L533）里串起来：

障碍怎么变成 target： getTargetObstacles（L662）对每个感知物体，用 polygon_utils 算它的多边形和自车轨迹的交点（collision_points）。只有==轨迹真的会穿过障碍==（有 collision_points）的物体才进 target_obstacles。这一步是整个触发的总开关：没交点 = 撞不上 = 直接忽略。

停车触发：generateStopTrajectory（planner_interface.cpp L86）

从 target_obstacles 里挑最近的停车障碍，算到碰撞点的弧长距离，减掉安全 margin 和车头偏移，得到该停的位置，再用 insertStopPoint（L196）把一个零速点插进轨迹。

里面有个关键的可行性检查 calcMinimumDistanceToStop（L75）——就是高中刹车距离公式：

$$d_{brake} = \frac{v_0^2}{2,|a_{min}|}$$

含义：以当前车速 $v_0$、用极限制动 $a_{min}$，最少要多少距离才能停下。如果障碍比这个距离还近（closest_obstacle_stop_dist < feasible_stop_dist，L186），说明==按极限制动也停不住==，会撞，于是置 will_collide_with_obstacle = true 发告警（L184-189）。这是个兜底的安全诊断，不是触发条件本身。

巡航触发：generateCruiseTrajectory + RSS 距离

对于要”跟”的障碍（前车），用 RSS（Responsibility-Sensitive Safety）模型算安全跟车距离，定义在 calcRSSDistance（planner_interface.hpp L127）：

$$d_{rss} = v_{ego}\cdot t_{idle} + \frac{v_{ego}^2}{2,|a_{ego}|} - \frac{v_{obj}^2}{2,|a_{obj}|} + margin$$

四项分别是：自车反应时间走的距离 + 自车制动距离 − 障碍自己的制动距离 + 余量。第三项减掉是因为前车也在减速，不需要给它留”完全停下”的整段距离。算出来的 $d_{rss}$ 用来定限速：跟车距离不够就降速，够就维持。输出是 VelocityLimit（限速指令），不是急停——和停车触发的区别就在这。

问题定义：自由空间规划

自由空间规划（freespace / parking planning）要解决：给定起点位姿 $(x_s, y_s, \theta_s)$、目标位姿 $(x_g, y_g, \theta_g)$ 与代价地图（costmap），在满足车辆运动学、且不与障碍碰撞的前提下，生成可行轨迹。

它与结构化道路规划的关键区别：

维度 结构化道路规划 自由空间规划
参考线 有车道中心线 / 路网路由 无明确参考线，近乎”白板”
状态空间 通常 $(s, d)$（纵向、横向） 完整 $(x, y, \theta)$，且常含正反向
运动学 小横向偏移，前向为主 大曲率、可能含倒车，非完整约束主导
避障 沿车道做横向避让 在开阔空间中绕开任意形状障碍
典型算法 Frenet 采样的 Lattice / EM planner Hybrid A*、RS / Dubins 曲线、RRT*

避障是自由空间规划的内在要求：与结构化道路”沿车道微调横向”不同，自由空间规划要在无参考线的开阔区域中主动绕开任意分布的障碍，因此”运动学可行 + 不碰障碍 + 路径合理”三者必须同时满足（详见后文避障设计）。

车辆采用阿克曼 / 自行车模型，最小转弯半径为硬约束：

$$
\dot x = v\cos\theta,\quad \dot y = v\sin\theta,\quad \dot\theta = \frac{v}{L}\tan\delta,\quad R_{\min} = \frac{L}{\tan\delta_{\max}}
$$

其中 $L$ 为轴距，$\delta_{\max}$ 为最大前轮转向角。RS 曲线、Dubins 曲线与 Hybrid A* 均建立在该模型之上，差异在于是否允许倒车（$v$ 是否可取负）——这正是 Dubins（仅前进）与 RS（前进+后退）的分水岭。

阿克曼底盘运动学模型

阿克曼转向结构是现代汽车的转向方式，可以解决汽车在转向时，由于左右转向轮转向半径不同所造成的左、右转向轮转角不同的问题。

先观察阿克曼运动模态简化之后的模型图。与差速运动相同，如果我们给四个车轮相同的速度（大小+方向），即可实现机器人的前进和后退。不同之处在于转弯，需要根据前端两个车轮的偏转方向，利用阿克曼转向几何来计算转弯半径。

其中 $V$ 为车辆速度，由油门 throttle、挡位 gear、刹车 brake 三个因素控制；$w$ 为车辆航向角速度 heading，由转向 steering 控制。

Simple Car Model

Simple Car Model 是指一种用于描述车辆运动的简化模型。具体来说，Simple Car Model 将车辆视为一个二维平面上的刚体，其原点位于车辆后轮的中心，x 轴沿着车辆主轴方向，与车辆运动方向一致。车辆的状态由位置坐标 $(x, y)$ 和姿态角 $\theta$ 表示。在 Simple Car Model 中，车辆的速度为 $s$，方向盘的转角为 $\alpha$，前轮和后轮中心的距离为 $L$。如果方向盘转角固定，车辆会在原地转圈，转圈的半径为 $L/\sin(\alpha)$。在非常短的时间内，车辆可以近似视为沿着后轮指向的方向前进；当方向盘转角趋于 0 时，车辆的运动方向与 x 轴一致。Simple Car Model 通过引入 Action 变量，假设车辆的运动速度和方向盘转角由这些变量控制，从而得到车辆的运动模型。这个模型适用于自动驾驶运动规划中的运动方程，可以描述车辆的向前或向后运动。

运动学方程如下：

$$
\begin{align*}
\dot{x} &= u_1 \cos\theta \
\dot{y} &= u_1 \sin\theta \
\dot{\theta} &= u_1 u_2
\end{align*}
$$

其中，$u_1 \in [-1, 1]$，$u_2 \in [-\tan\phi_{max}, \tan\phi_{max}]$。当 $u_1 = 1$ 时表示车辆向前运动；$u_1 = -1$ 时表示车辆向后运动。$\phi_{max}$ 代表车辆允许的最大转向角。

关键观察：$u_1$ 的取值范围决定规划性质——

  • Dubins 曲线：限定 $u_1 = +1$（仅前进），只在前向中求最短；
  • Reeds-Shepp 曲线：允许 $u_1 \in {-1, +1}$（前进+后退），在双向中求最短。

对东营下料区这种开阔、可一次前向 U 型掉头的场景，$u_1 \equiv +1$ 已足够，Dubins 即可胜任，无需承担倒车带来的换向与控制复杂度。

Reeds-Shepp 曲线

原理

Reeds & Shepp（1990）证明：在最小转弯半径 $R_{\min}$ 约束、允许前进与后退的汽车，从任一起点位姿到任一目标位姿的最短路径，必然属于一个有限的路径类型集合，每种类型均可由”圆弧段（左转 L / 右转 R）+ 直线段（S）+ 方向符号（前进 + / 后退 −）”解析拼接得到。因此 RS 曲线不需要迭代搜索：归一化目标到原点参考系，枚举有限类候选、计算各段长度、取最短且无符号矛盾者即可。

数学模型要点

  • 路径由最多 3 段组成，每段为圆弧或直线，带前进 / 后退方向；
  • 48 种组合经对称性归并为 12 类等价路径族（如 LRL、LSL、LRSL 等）；
  • 全程曲率取值集合为 ${0,\ \pm 1/R_{\min}}$（bang-bang 控制）；
  • 路径长度可由起止位姿差解析求出，时间复杂度 $O(1)$。

Reeds-Shepp 曲线

Reeds-Shepp 曲线是从 46 条曲线中选出最短的那条，这 46 条曲线如下图所示。（最开始的论文认为最短的曲线一定在 48 条曲线中，后人研究发现有两条曲线不会是最短的，所以搜索范围减小到 46 条。）

优点

  • 解析最优：无搜索、无调参，毫秒级；
  • 运动学严格可行：天然满足最小转弯半径；
  • 可作”精确命中”工具：常用于 Hybrid A* 的解析扩展（RS shot）与控制层末段对齐；
  • 天然支持倒车：适用于必须倒车的紧凑对接（如倒车入库）。

局限

  • 不考虑障碍：只保证起止位姿间运动学最短，中间可能穿越障碍 / 车位边界，必须外挂碰撞检测；
  • 曲率不连续：段间曲率阶跃，实际跟踪转向急变，通常需后接平滑（样条 / 共轭梯度优化）；
  • 单一目标位姿：对目标朝向容差不友好；
  • 倒车带来换向：前进↔后退切换需停车换挡，控制与执行更复杂。

适用场景

起止位姿接近、中间开阔时的短距精确对接；作为更大规划框架（Hybrid A*、采样规划器）内部的局部衔接段；车位内”最后一米”精确入位；需要倒车的紧凑对接（东营储存库倒车入库类场景）。

Dubins 曲线（前进向特例）

**引入动机：储存库（下料区）场地足够开阔，车辆可”一个 U 型直接掉头、不用倒车”。此时路径只需在前进向中求最短即可，Dubins 曲线正是该问题的解析最优解，无需引入倒车及其带来的换向停车、单向分段输出等复杂度。

原理

Dubins（1957）证明：仅允许前进、最小转弯半径为 $R_{\min}$ 的汽车，在任一起止位姿间的最短路径，必然属于 6 种路径类型之一。它是 RS 曲线的前向特例——将 Simple Car Model 中的速度变量限定为 $u_1 \equiv +1$（仅前进）：

$$
\dot{x} = \cos\theta,\quad \dot{y} = \sin\theta,\quad \dot{\theta} = u_2,\qquad u_1 = 1,\ \ u_2 \in [-\tan\phi_{max},\ \tan\phi_{max}]
$$

全程曲率取值集合同为 ${0,\ \pm 1/R_{\min}}$，但所有段均为前进方向。

数学模型要点

  • 6 种候选类型，分两组：
    • CSC 型（圆弧-直线-圆弧）：LSL、LSR、RSL、RSR；
    • CCC 型（三段圆弧）：RLR、LRL；
  • 从 6 种中取长度最短者，时间复杂度 $O(1)$；
  • 与 RS 的关系：RS = Dubins 的”双向推广”（允许 $u_1=\pm1$​，候选由 6 类扩展到 48/46 类）。

图：LRL示意图

优点

  • 解析最优：无搜索、无调参，毫秒级；
  • 运动学严格可行：天然满足最小转弯半径；
  • 纯前向、无换向：全程不倒车 → 速度方向连续、无需停车换挡、跟踪更平滑，也规避了 Autoware 单向分段输出在换向点的执行复杂度；
  • 最简单：6 类候选，计算与实现都比 RS 更轻。

局限

  • 不考虑障碍：同 RS，需外挂碰撞检测；
  • 曲率不连续：段间阶跃，常需后处理平滑；
  • 不能倒车：无法完成必须倒车的紧凑机动（如贴墙倒车入库、三点式掉头、狭窄泊位入位）——这正是它在储存库（下料区）**开阔掉头”够用”、但泊位入位需倒车时”不够用”**的原因。

适用场景

开阔场地、仅靠前进即可完成换向/对接的机动。典型即东营储存库（下料区）的 U 型掉头：场地足够大，一个半径 ≥ $R_{\min}$ 的前向半圆（或 LRL/RLR 三弧）即可 180° 换向，根本不需要倒车。此外也适合作为前向规划器（如 use_back=false 的 Hybrid A*）的解析扩展（Dubins shot）。

图：储存库（下料区）开阔机动与泊位空间示意，U 型掉头可在前向完成

下料区 U 型掉头的选型含义：在该子任务上，”前向最优”的 Dubins 思路比”双向最优”的 RS 思路更合适——少一类复杂度（不倒车）、执行更顺滑。在 Autoware 中对应将 astar.use_back 置为 false（并/或提高 reverse_weight），使规划器退化为前向 Hybrid A*（Dubins 启发式 + 前向 shot）。

Hybrid A*

原理

Hybrid A*（Dolgov, Thrun, Montemerlo, Diebel, 2008，源自 DARPA Urban Challenge “Junior”）是在连续三维状态空间 $(x, y, \theta)$ 上的 A* 搜索。区别于经典网格 A*：节点状态是连续位姿，但用网格单元 + 航向离散桶判重（closed set），既保证运动学可行性，又控制搜索规模。

状态扩展与运动原语

  • 状态：$(x, y, \theta)$，部分实现再加转向角 $\phi$ 成为 4D；
  • 运动原语：若干离散转向角（最大左、最大右、中间档）× 前进 / 后退，按固定步长积分自行车模型生成后继节点；
  • 碰撞检测：沿新扩展段用车辆足印（数个圆 / 矩形 + 安全裕度）在 costmap 上检查——这是 Hybrid A* 区别于 RS/Dubins 的核心：原生考虑障碍。

代价函数

$$
g(n) = \sum \bigl(\text{长度}\cdot w_{\text{len}} + \text{转向变化}\cdot w_{\text{smooth}} + \text{倒车}\cdot w_{\text{rev}} + \text{换向}\cdot w_{\text{dir}} + \text{贴近障碍}\cdot w_{\text{obs}}\bigr)
$$

通过权重在”少倒车 / 少换向 / 远离障碍 / 平滑”之间取舍。其中”贴近障碍”项直接驱动路径主动远离障碍。

双启发式（关键设计）

同时维护两个启发函数并取较大者，保证 admissible 又足够有引导力：

  1. 非完整约束、无障碍启发：到目标的 RS（或 Dubins）距离，对若干候选目标航向取最小 —— 引导搜索沿运动学可行方向收敛；
  2. 完整约束、含障碍启发：在 2D costmap 上用 Dijkstra 预计算的带障碍最短距离 —— 在遇墙 / 死胡同处推高 h 值，避免盲目撞向障碍。

启发式里”RS 或 Dubins 距离”的选择可与场景方向策略对齐：前向场景用 Dubins 距离，需倒车场景用 RS 距离。

Reed-Shepp 解析扩展（RS Shot）

当节点进入目标邻域时，直接向目标射一条 RS（或 Dubins）曲线，若碰撞空闲即作为解析解接上。此步大幅加速收敛并保证终点精确命中，是 Hybrid A* 工程有效性的核心来源。

优点

运动学可行 + 考虑障碍；终点靠解析 shot 精确命中；代价可调，适配不同诉求（含显式的避障/平滑/少倒车偏好）。

局限

曲率仍非完全连续（段间阶跃，常需后处理平滑）；分辨率与参数敏感；贴近障碍时易失败 / 耗时长；默认只给路径，速度需下游补全；对快速动态障碍的响应依赖重规划与局部层（见避障设计）。

  • 曲率连续性：本质为离散转向 Hybrid A*，段间曲率阶跃；
  • max_turning_ratio = 0.5：搜索只用半转向范围，是窄空间规划失败或路径偏长的隐性原因；
  • 仅恒速输出：速度剖面需下游模块补全；
  • 动态避障依赖下游：freespace_planner 面向静态/半静态规划，快速动态障碍需局部执行层处理。

自由空间规划中的避障设计

本节回答带教提出的问题：自由规划情况下如何考虑避障。

障碍分类（结合东营厂区）

类别 东营厂区具体对象 特征
静态 储存库 1–6 号、高架库结构、储罐（附图椭圆体）、管道、阀门、设备基座、护栏/路缘、建筑外墙 位置固定，可固化进静态 costmap
半静态 临时堆放物料、托盘、停靠的其他车辆 缓慢变化，靠点云/感知更新
动态 其他无人物流车、巡检/作业人员、叉车 实时运动，需短周期感知与响应

避障机制（Autoware 链路）

  1. costmap 表示：obstacle_threshold 区分占用/空闲；inflation（膨胀）层在障碍周围绕出代价梯度（离障碍越近代价越高），形成”软边界”。
  2. 车辆足印碰撞检测 + 安全裕度：沿候选扩展段用车辆足印（多圆/矩形）+ vehicle_shape_margin_m 外扩后在 costmap 上检查，保证整条扫掠包络不碰障碍。
  3. 远离障碍代价：astar.obstacle_distance_weight 把”贴近障碍”计入代价，主动把路径推向远离障碍的走廊。
  4. 遇障重规划：replan_when_obstacle_found 在发现新障碍、或 replan_when_course_out 在跑偏时触发重新规划——这是 freespace 规划应对环境变化的主要入口。

分层避障职责（关键）

自由空间场景下，避障是分层完成的，需明确分工，避免职责真空：

层级 模块 职责 响应对象
全局机动规划 freespace_planner（Hybrid A*） 输出可行且不碰静态/半静态障碍的机动轨迹；带重规划 静态/半静态障碍、环境变化
局部实时执行 surround_obstacle_checker / obstacle_stop_planner / obstacle_avoidance_planner / obstacle_cruise_planner 实时动态避障：减速、停车、局部绕行 动态障碍、突发入侵
换向点协同 单向分段输出 + 局部执行链 在前进↔后退换向停车点做避障检查与下一段触发 多段机动的衔接

即：freespace_planner 保证”规划出来的路本身不撞静态障碍”，但”行驶途中突然出现的动态障碍”由局部执行层实时处理。规划与控制必须衔接好（对应 W2 的局部执行链走读）。

储存库（下料区）特定避障要点

  • 对接设施即障碍：高架库（上料区，南面上料）与储存库（下料区）泊位均需精确对接，周边的对接设备/月台/管道等应标记为高代价/禁入区——对接点为 goal，周边设施规划绕开后精确对位；
  • 人工下料区的人员安全：储存库北面为人工下料，有人员活动，该侧需更高安全裕度与低速限制，并在 costmap 中对人工作业区做膨胀/禁入处理；
  • 多车协同：储存库（下料区）与转运线路可能存在多台物流车并行作业，需考虑区域互斥/时序（驶入下料区前确认空闲），属 W3/W4 议题；
  • 适度增大裕度：厂区设施密集，建议在该区域适度上调 vehicle_shape_margin_m 与 astar.obstacle_distance_weight，换取安全性（代价是窄处的规划成功率，需平衡）；
  • 开阔区可前向绕行：储存库（下料区）开阔，遇障时前向绕行（Dubins/Hybrid A* 前向）通常可行，无需倒车脱困。

风险与局限

  • freespace_planner 为静态/半静态规划，对快速动态障碍响应不足，强依赖局部层；
  • 贴近障碍时规划成功率下降（autoware #8699），下料区设备旁对接需重点验证；
  • costmap 质量（点云完整性、动态物体滤除）直接决定避障可靠性，需在 W2 确认感知→costmap 链路。

  • 基线在东营标准车位 / 下料区掉头场景下验收不达标（轨迹穿越边界、曲率不连续致跟踪误差超标、末点停位误差超标）；
  • 贴近障碍（下料区设备旁对接）规划成功率不稳；
  • U 型掉头多段衔接在局部执行层切换抖动 / 失败；
  • 局部动态避障对厂区多车场景响应不足。

  • 自由空间规划的核心矛盾是”运动学可行性 ↔ 避障 ↔ 计算效率“三角。Dubins 与 RS 曲线在运动学最优与效率上极致，但不避障、曲率不连续，差异在”是否允许倒车”；Hybrid A* 用连续状态搜索 + 双启发式 + 解析 shot 取得平衡并原生考虑障碍；Autoware 模块把后者工程化并接入完整 Planning 链路。

  • 地图需 Lanelet2（转运线路路由）+ freespace costmap（机动区规划）双图协同，在出口/入口/泊位 handover 衔接、统一 map frame、覆盖完整机动包络。

  • 重点关注单向分段输出、max_turning_ratio=0.5、曲率连续性、贴近障碍成功率等特性，作为 W3/W4 是否自研增强的决策依据。

全局路径规划算法学习笔记

全局路径规划算法学习笔记（A* / Hybrid A* / RRT / RRT*）

本文聚焦四个最经典的全局路径规划算法：A*、Hybrid A*（混合 A*）、RRT、RRT*。

先把全局图景搭起来（思维导图）

A* / Hybrid A* 属于”基于搜索”，RRT / RRT* 属于”基于采样”。前者把空间切成格子或用运动基元一格一格扩，后者在空间里随机撒点往目标长。两条路线各有适用场景，下面逐个拆。

一些贯穿全文的概念

概念 含义
状态 / 节点 State 规划的基本单元。A* 里是栅格坐标，Hybrid A* 里是位姿 (x, y, θ)，RRT 里是连续点。
完备性 Completeness “只要解存在，算法一定能找到”。A* 是完备的，RRT 是”概率完备”（时间够长概率趋于1）。
最优性 Optimality “找到的解是不是代价最小的”。A*（在 h 满足条件时）最优；RRT 不最优；RRT* “渐进最优”。
完备 vs 最优 完备保证”能找到”，最优保证”找到的是最好的”。两者独立。
代价函数 Cost function 衡量一条路/一个状态有多”贵”的函数，搜索类算法靠它排序、采样类靠它重连。
启发函数 Heuristic h(n) 对”剩余距离”的估计，只搜索类（A*/Hybrid A*）用，用来优先扩展有希望的方向。
碰撞检测 Collision check 判断一段路径/一个状态是否撞障碍。所有算法都要做，往往是性能瓶颈。

完备性、最优性是评价规划算法的两把尺子，下面的对比会反复用到。

A* 算法

Dijkstra 和贪心最好优先的结合

  • Dijkstra：只看已经走了多远 g(n)，向四面八方均匀扩展。保证最短，但慢，会探索很多没用的方向。
  • 贪心最好优先（Greedy Best-First）：只看估计离终点还有多远 h(n)，猛冲目标。快，但可能被障碍带偏，不保证最短。
  • A*：两者相加，f(n) = g(n) + h(n)。既考虑已走代价，又用启发值引导方向。当 h 是”可采纳的（admissible，即从不高估）”时，A* 保证最优。

这就是 A* 的全部灵魂：用一个加权和给每个节点打分，永远优先选择扩展分最低的节点。

三个核心量

h 的选择（栅格地图常用）：

  • 4 连通 → 曼哈顿距离 |dx| + |dy|
  • 8 连通 → 对角线距离 max(|dx|,|dy|) + (√2−1)·min(|dx|,|dy|)
  • 一般情况 → 欧氏距离 √(dx²+dy²)

对 8 连通栅格，对角线距离最贴合实际可走路径，是最常用的选择。h 必须不超过真实最短距离，否则最优性失效。

特性 四连通 (4-Connected) 八连通 (8-Connected)
可移动方向 4 个（上下左右） 8 个（包含对角线）
推荐的 $h$ 函数 曼哈顿距离 对角线距离 / 切比雪夫距离
路径效果 锯齿状折线，比较生硬 更平滑，允许走捷径
计算复杂度 每次探寻 4 个邻居，计算稍快 每次探寻 8 个邻居，计算量稍大

两个关键数据结构

  • Open List（开放表）：已发现、还没扩展的节点。通常用最小堆/优先队列，按 f 值排序，取最小 O(log n)。
  • Closed List（关闭表）：已经扩展过的节点。用哈希集合，O(1) 查重。

算法链路（执行流程）

代码实现

输入 / 输出 / 代价函数

  • 输入：栅格地图（含障碍）、起点、终点、启发函数 h
  • 输出：从起点到终点的一条最短路径（节点序列），或”无解”
  • 代价函数：
    • 节点级：f(n) = g(n) + h(n)，g 是累积步长代价（直走=1，斜走=√2）
    • 路径级：整条路径 g 的和，即总长度
    • 想加偏好（如远离墙、少转弯），在 cost(n,m) 里加权即可

优缺点小结

优点 缺点
完备 + 最优（h 可采纳时） 路径是栅格折线，车辆无法直接跟踪
思想简单、实现成熟 2D 还行，维度一高（3D+）就”维度灾难”，爆炸
启发值可控，可调节速度/质量 不考虑运动学（朝向、最小转弯半径）

Hybrid A*（混合 A*）

为什么要有 Hybrid A*

普通 A* 的路径长这样：在栅格中心之间走折线、走 45°。一辆真实的车（阿克曼转向，有最小转弯半径）根本走不出来。如果直接拿 A* 路径给控制器，要么跟踪不上，要么得做大量后处理（平滑、拟合），还可能拟合失败。

Hybrid A* 的目标：直接在车辆的”位姿空间”里搜索，让搜出来的路径本身就能被车开出来。

核心改动：状态从离散栅格 → 连续位姿

A* Hybrid A*
状态 栅格 (i, j) 连续位姿 (x, y, θ)，θ 是车头朝向
邻居生成 上下左右/对角 用车辆运动学前进一步（运动基元 motion primitive）
路径形状 栅格折线 真实可行驶轨迹（圆弧/RS 曲线段拼接）

连续空间怎么搜索？——3D 栅格去重

纯连续位姿有无限多个，没法搜。Hybrid A* 的巧思：

  • 把连续位姿 (x, y, θ) 投影到一个 3D 离散栅格（x 分箱、y 分箱、θ 分箱）。
  • 每个 3D cell 里只保留一个状态（通常是代价最低的那个，或第一个到达的）。
  • 这样搜索空间是有限的（分辨率完备），但每个状态本身又是连续的真实位姿——路径连续平滑，但搜索规模可控。

一句话：用离散栅格做”去重/记忆”，用连续状态做”路径生成”。这就是 “Hybrid（混合）” 二字的含义——离散 + 连续的混合。

启发函数要升级

普通欧氏距离在 Hybrid A* 里太弱（因为还要考虑朝向和不能横移），扩展会爆炸。常用更强的启发：

  1. 2D A* / Dijkstra 距离：先在 2D（忽略 θ、允许任意方向移动）上跑一遍，拿到每个格子到目标的距离，作为 h。这已经比欧氏强很多。
  2. Reeds-Shepp 距离：直接算”如果没障碍，从当前位姿到目标的最短 RS 曲线长度”。更强，但要配合碰撞检查裁剪。

工程上常把两者结合，甚至取多种启发的最大值，兼顾引导力和可采纳性。

Reeds-Shepp 加速：这是 “A* RS” 的 RS 部分

光靠运动基元一格一格扩，最后可能永远”差一点点”对不上目标的精确朝向 θ。解决办法：

  • 每次扩展出一个节点后，尝试用 Reeds-Shepp 曲线直接连到目标。
  • 如果这条 RS 曲线无碰撞，那就直接接上、结束搜索。
  • 这让算法能精确收敛到目标位姿，大幅减少靠近目标时的无谓扩展。

Reeds-Shepp 曲线：给定两个位姿，在”车能前进、能后退”的前提下，存在解析的最短路径（由若干段圆弧+直线组合，公式可查表）。能前进能后退 = 适合泊车这种需要揉库的场景。
Dubins 曲线：只能前进不能后退的对应物，适合单向行驶。

RRT（Rapidly-exploring Random Tree，快速探索随机树）

直觉：往随机方向长树

基于搜索的方法在高维空间（比如机械臂 6 关节 + 障碍）会爆炸——栅格数随维度指数增长。RRT 的思路完全不同：不切格子，直接在空间里随机撒点，让一棵树朝随机点生长。

为什么”快速探索”？因为均匀随机撒点，树会均匀地向四面八方伸展，自然就快速覆盖整个空间，不会卡在一个角落。它对维度不敏感，6 维、10 维照样跑。

这个”不最优”正是 RRT* 要补的短板。

RRT*（Optimal RRT，最优 RRT）

RRT 的问题在哪

RRT 一个节点连上父节点后，就再也不改了。哪怕后面发现”从另一个节点连过来更近”，它也不会重连。所以路径往往绕远、冗长。

RRT* 的核心改进：每次加新节点时，不仅给新节点找最好的爸爸，还顺便帮邻居们重新认爸爸。 这样随着采样增多，整棵树越来越”顺”，路径越来越短。

两个关键操作

设新节点 q_new 已经生成，在它周围画一个半径 r 的邻域，邻域内的节点集合记为 Q_near。

① ChooseParent（重新选父）——给 q_new 找最便宜的爹

→ q_new 一开始就连上最优父节点，而不是最近的那个。

② Rewire（重布线）——帮邻居们换个更便宜的爹

→ 老节点也能受益于新节点，整棵树持续优化。

渐近最优性（Asymptotic Optimality）：当采样数 → ∞，路径代价 → 全局最优。代价是：每次迭代要多做很多碰撞检测和距离计算，单步比 RRT 慢不少。

邻域半径 r 有理论公式（随采样数 n 减小），保证渐近最优性的同时控制计算量。工程上常取 r = γ · (log n / n)^(1/d)，d 是空间维度。

串回你的主线：freespace planning 与 “A* RS”

现在把四个算法和你的学习目标接起来：

freespace planning 是什么

“freespace planning” 通常指在自由空间（无障碍的连续空间）里做几何/运动学规划，自动驾驶里特别用于低速泊车、掉头、揉库这类场景——需要精确控制终点位姿（车要停进车位，朝向也得对）。

“A* RS” 到底是什么

你看到的 “A + RS”* 基本就是 Hybrid A* 配 Reeds-Shepp 曲线，即本文第 3 节那套：

  • A* 部分：用 A* 那套 f=g+h、Open/Closed、优先队列的搜索框架，但状态是连续位姿、用运动基元扩展（= Hybrid A*）。
  • RS 部分：
    1. 运动基元用 RS 曲线段（而非简单圆弧），更贴合”能前进能后退”的车；
    2. 接近目标时用 RS 曲线”射靶”，解析地精确收敛到目标位姿。

所以理解了第 3 节，你的 “A* RS” 也就通了 80%。剩下的 20% 是工程调参：启发函数选哪种、代价权重怎么定、3D 栅格分辨率多细。

为什么泊车用 RS 而不是 Dubins

Reeds-Shepp Dubins
前进 ✅ ✅
后退 ✅ ❌
适合 泊车/揉库（要换挡） 单向巡航/无人机

泊车场景车要反复前进后退对准车位，所以 RS 更合适——这也是 freespace planning 几乎都用 RS 的原因。
