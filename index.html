全局路径规划算法学习笔记（A* / Hybrid A* / RRT / RRT*）

本文聚焦四个最经典的全局路径规划算法：A*、Hybrid A*（混合 A*）、RRT、RRT*。

先把全局图景搭起来（思维导图）

A* / Hybrid A* 属于”基于搜索”，RRT / RRT* 属于”基于采样”。前者把空间切成格子或用运动基元一格一格扩，后者在空间里随机撒点往目标长。两条路线各有适用场景，下面逐个拆。

一些贯穿全文的概念

概念 含义
状态 / 节点 State 规划的基本单元。A* 里是栅格坐标，Hybrid A* 里是位姿 (x, y, θ)，RRT 里是连续点。
完备性 Completeness “只要解存在，算法一定能找到”。A* 是完备的，RRT 是”概率完备”（时间够长概率趋于1）。
最优性 Optimality “找到的解是不是代价最小的”。A*（在 h 满足条件时）最优；RRT 不最优；RRT* “渐进最优”。
完备 vs 最优 完备保证”能找到”，最优保证”找到的是最好的”。两者独立。
代价函数 Cost function 衡量一条路/一个状态有多”贵”的函数，搜索类算法靠它排序、采样类靠它重连。
启发函数 Heuristic h(n) 对”剩余距离”的估计，只搜索类（A*/Hybrid A*）用，用来优先扩展有希望的方向。
碰撞检测 Collision check 判断一段路径/一个状态是否撞障碍。所有算法都要做，往往是性能瓶颈。

完备性、最优性是评价规划算法的两把尺子，下面的对比会反复用到。

A* 算法

Dijkstra 和贪心最好优先的结合

  • Dijkstra：只看已经走了多远 g(n)，向四面八方均匀扩展。保证最短，但慢，会探索很多没用的方向。
  • 贪心最好优先（Greedy Best-First）：只看估计离终点还有多远 h(n)，猛冲目标。快，但可能被障碍带偏，不保证最短。
  • A*：两者相加，f(n) = g(n) + h(n)。既考虑已走代价，又用启发值引导方向。当 h 是”可采纳的（admissible，即从不高估）”时，A* 保证最优。

这就是 A* 的全部灵魂：用一个加权和给每个节点打分，永远优先选择扩展分最低的节点。

三个核心量

h 的选择（栅格地图常用）：

  • 4 连通 → 曼哈顿距离 |dx| + |dy|
  • 8 连通 → 对角线距离 max(|dx|,|dy|) + (√2−1)·min(|dx|,|dy|)
  • 一般情况 → 欧氏距离 √(dx²+dy²)

对 8 连通栅格，对角线距离最贴合实际可走路径，是最常用的选择。h 必须不超过真实最短距离，否则最优性失效。

特性 四连通 (4-Connected) 八连通 (8-Connected)
可移动方向 4 个（上下左右） 8 个（包含对角线）
推荐的 $h$ 函数 曼哈顿距离 对角线距离 / 切比雪夫距离
路径效果 锯齿状折线，比较生硬 更平滑，允许走捷径
计算复杂度 每次探寻 4 个邻居，计算稍快 每次探寻 8 个邻居，计算量稍大

两个关键数据结构

  • Open List（开放表）：已发现、还没扩展的节点。通常用最小堆/优先队列，按 f 值排序，取最小 O(log n)。
  • Closed List（关闭表）：已经扩展过的节点。用哈希集合，O(1) 查重。

算法链路（执行流程）

代码实现

输入 / 输出 / 代价函数

  • 输入：栅格地图（含障碍）、起点、终点、启发函数 h
  • 输出：从起点到终点的一条最短路径（节点序列），或”无解”
  • 代价函数：
    • 节点级：f(n) = g(n) + h(n)，g 是累积步长代价（直走=1，斜走=√2）
    • 路径级：整条路径 g 的和，即总长度
    • 想加偏好（如远离墙、少转弯），在 cost(n,m) 里加权即可

优缺点小结

优点 缺点
完备 + 最优（h 可采纳时） 路径是栅格折线，车辆无法直接跟踪
思想简单、实现成熟 2D 还行，维度一高（3D+）就”维度灾难”，爆炸
启发值可控，可调节速度/质量 不考虑运动学（朝向、最小转弯半径）

Hybrid A*（混合 A*）

为什么要有 Hybrid A*

普通 A* 的路径长这样：在栅格中心之间走折线、走 45°。一辆真实的车（阿克曼转向，有最小转弯半径）根本走不出来。如果直接拿 A* 路径给控制器，要么跟踪不上，要么得做大量后处理（平滑、拟合），还可能拟合失败。

Hybrid A* 的目标：直接在车辆的”位姿空间”里搜索，让搜出来的路径本身就能被车开出来。

核心改动：状态从离散栅格 → 连续位姿

A* Hybrid A*
状态 栅格 (i, j) 连续位姿 (x, y, θ)，θ 是车头朝向
邻居生成 上下左右/对角 用车辆运动学前进一步（运动基元 motion primitive）
路径形状 栅格折线 真实可行驶轨迹（圆弧/RS 曲线段拼接）

连续空间怎么搜索？——3D 栅格去重

纯连续位姿有无限多个，没法搜。Hybrid A* 的巧思：

  • 把连续位姿 (x, y, θ) 投影到一个 3D 离散栅格（x 分箱、y 分箱、θ 分箱）。
  • 每个 3D cell 里只保留一个状态（通常是代价最低的那个，或第一个到达的）。
  • 这样搜索空间是有限的（分辨率完备），但每个状态本身又是连续的真实位姿——路径连续平滑，但搜索规模可控。

一句话：用离散栅格做”去重/记忆”，用连续状态做”路径生成”。这就是 “Hybrid（混合）” 二字的含义——离散 + 连续的混合。

启发函数要升级

普通欧氏距离在 Hybrid A* 里太弱（因为还要考虑朝向和不能横移），扩展会爆炸。常用更强的启发：

  1. 2D A* / Dijkstra 距离：先在 2D（忽略 θ、允许任意方向移动）上跑一遍，拿到每个格子到目标的距离，作为 h。这已经比欧氏强很多。
  2. Reeds-Shepp 距离：直接算”如果没障碍，从当前位姿到目标的最短 RS 曲线长度”。更强，但要配合碰撞检查裁剪。

工程上常把两者结合，甚至取多种启发的最大值，兼顾引导力和可采纳性。

Reeds-Shepp 加速：这是 “A* RS” 的 RS 部分

光靠运动基元一格一格扩，最后可能永远”差一点点”对不上目标的精确朝向 θ。解决办法：

  • 每次扩展出一个节点后，尝试用 Reeds-Shepp 曲线直接连到目标。
  • 如果这条 RS 曲线无碰撞，那就直接接上、结束搜索。
  • 这让算法能精确收敛到目标位姿，大幅减少靠近目标时的无谓扩展。

Reeds-Shepp 曲线：给定两个位姿，在”车能前进、能后退”的前提下，存在解析的最短路径（由若干段圆弧+直线组合，公式可查表）。能前进能后退 = 适合泊车这种需要揉库的场景。
Dubins 曲线：只能前进不能后退的对应物，适合单向行驶。

RRT（Rapidly-exploring Random Tree，快速探索随机树）

直觉：往随机方向长树

基于搜索的方法在高维空间（比如机械臂 6 关节 + 障碍）会爆炸——栅格数随维度指数增长。RRT 的思路完全不同：不切格子，直接在空间里随机撒点，让一棵树朝随机点生长。

为什么”快速探索”？因为均匀随机撒点，树会均匀地向四面八方伸展，自然就快速覆盖整个空间，不会卡在一个角落。它对维度不敏感，6 维、10 维照样跑。

几个关键点：

  • Steer 函数：决定”怎么走这一步”。最简单是直线插值；要考虑运动学（如车辆）就用 RS/Dubins 段。
  • goal bias：纯随机效率低，所以通常以 5%~10% 概率直接把目标点当 q_rand，引导树往目标长。

输入 / 输出 / 代价函数

  • 输入：采样空间、起点、目标区域、步长、碰撞检测函数
  • 输出：一条可行路径（注意：不保证短、不保证平滑，往往很曲折）
  • 代价函数：RRT 本身基本不显式优化代价，主要靠”局部路径长度”做 Steer 和 Nearest。路径质量差，需要后处理（shortcut 抄近路、smoothing 平滑）。

优缺点小结

优点 缺点
高维空间表现好，不炸 概率完备（非确定完备）
实现简单、对复杂障碍适应强 路径不最优，曲折难走
速度快，适合”先找个解” 单次运行结果随机，不稳定

这个”不最优”正是 RRT* 要补的短板。

RRT*（Optimal RRT，最优 RRT）

RRT 的问题在哪

RRT 一个节点连上父节点后，就再也不改了。哪怕后面发现”从另一个节点连过来更近”，它也不会重连。所以路径往往绕远、冗长。

RRT* 的核心改进：每次加新节点时，不仅给新节点找最好的爸爸，还顺便帮邻居们重新认爸爸。 这样随着采样增多，整棵树越来越”顺”，路径越来越短。

两个关键操作

设新节点 q_new 已经生成，在它周围画一个半径 r 的邻域，邻域内的节点集合记为 Q_near。

① ChooseParent（重新选父）——给 q_new 找最便宜的爹

→ q_new 一开始就连上最优父节点，而不是最近的那个。

② Rewire（重布线）——帮邻居们换个更便宜的爹

→ 老节点也能受益于新节点，整棵树持续优化。

渐近最优性（Asymptotic Optimality）：当采样数 → ∞，路径代价 → 全局最优。代价是：每次迭代要多做很多碰撞检测和距离计算，单步比 RRT 慢不少。

邻域半径 r 有理论公式（随采样数 n 减小），保证渐近最优性的同时控制计算量。工程上常取 r = γ · (log n / n)^(1/d)，d 是空间维度。

  • 代价函数：这里代价函数很关键——cost(节点) 是从根到该节点的累积路径长度（或加权代价），ChooseParent 和 Rewire 都是在最小化它。

优点 缺点

概率完备 理论最优 ≠ 工程上能等到最优

怎么记

  • 想要确定 + 最优 + 低维 → A*
  • 想要确定 + 路径能直接开 + 3D 位姿 → Hybrid A*（你的 freespace planning 主力）
  • 想要快速 + 高维 + 不在乎最优 → RRT
  • 想要高维 + 路径质量好 → RRT*

串回你的主线：freespace planning 与 “A* RS”

现在把四个算法和你的学习目标接起来：

freespace planning 是什么

“freespace planning” 通常指在自由空间（无障碍的连续空间）里做几何/运动学规划，自动驾驶里特别用于低速泊车、掉头、揉库这类场景——需要精确控制终点位姿（车要停进车位，朝向也得对）。

“A* RS” 到底是什么

你看到的 “A + RS”* 基本就是 Hybrid A* 配 Reeds-Shepp 曲线，即本文第 3 节那套：

  • A* 部分：用 A* 那套 f=g+h、Open/Closed、优先队列的搜索框架，但状态是连续位姿、用运动基元扩展（= Hybrid A*）。
  • RS 部分：
    1. 运动基元用 RS 曲线段（而非简单圆弧），更贴合”能前进能后退”的车；
    2. 接近目标时用 RS 曲线”射靶”，解析地精确收敛到目标位姿。

所以理解了第 3 节，你的 “A* RS” 也就通了 80%。剩下的 20% 是工程调参：启发函数选哪种、代价权重怎么定、3D 栅格分辨率多细。

为什么泊车用 RS 而不是 Dubins

Reeds-Shepp Dubins
前进 ✅ ✅
后退 ✅ ❌
适合 泊车/揉库（要换挡） 单向巡航/无人机

泊车场景车要反复前进后退对准车位，所以 RS 更合适——这也是 freespace planning 几乎都用 RS 的原因。

第九章：自动驾驶车辆的离线地图构建

点云建图的流程

上图是整个建图流程：

  1. 从给定的ROS数据包中解出IMU、RTK和激光雷达数据。使用IMU与激光雷达数据组成LIO系统。
  2. 大部分自动驾驶车辆会携带RTK设备或者组合导航设备。按照RTK或者组合导航的位姿给每个关键帧赋一个RTK位姿，作为观测。
  3. 使用LIO作为相邻帧运动观测，使用RTK位姿作为绝对坐标观测，优化整条轨迹并且判定各关键帧的RTK有效性。这是第一轮优化。
  4. 在上一步的基础上对地图进行回环检测，可以使用简单的基于欧氏距离的检测，使用NDT或者常见的配准算法计算相对位姿。
  5. 再把信息放到统一的位姿图中进行优化，消除累计误差带来的重影，同时移除失效区域。这是第二轮优化。确定位姿之后，可以按照这些位姿到处点云地图。为了方便地图加载和查看，可以对点云进行切片处理。处理之后的点云可以用来进行高精度定位或者地图标注。

前端实现

  1. 将ROS数据包中的RTK数据提取出来，放在RTK消息队列中，按照采集时间排序
  2. 用第一个有效的RTK数据作为地图原点，用其他的RTK读书减去原点，作为RTK位置观测
  3. 用IMU和激光雷达输出数据运行LIO，按照距离和角度阈值抽取关键帧，再保存关键帧结果。抽取关键帧时，也会从LIO中获取它的位姿和点云

后续的建图流程可以从前端的结果出发，不必解析ROS数据包。先使用RTK位姿和LIO位姿进行联合优化，在此基础上进行回环检测，消除累计误差。

后端位姿图优化和异常值检验

我们希望最终优化的轨迹能够考虑到各种传感器的输入，因此后端图优化需要包含以下因子：

  1. RTK因子。这是一种绝对的位姿观测数据。
  2. 雷达里程计信息，使用前面介绍的激光雷达里程计或者LIO作为局部连续性约束
  3. 回环因子，执行回环检测就加载回环检测的相关信息，把它们放入图优化中。

回环检测

点云地图的质量很大程度上取决于回环检测是，设计回环检测的步骤如下：

  1. 遍历第一轮优化轨迹中的关键帧。对每两个在空间上相隔较近，但是在时间上存在一定距离的关键帧进行一次回环检测。
  2. 对于每个关键帧对，使用Scan to Map 的配准方法，在关键帧附近抽取一定数量的点云作为子地图，然后对扫描和子地图进行配准。
  3. 由于每个检查点的计算都是独立的，使用并发编程让第二步能够在机器上并发执行，加快速度
  4. 配准的实际执行由NDT完成，使用NDT的分值作为回环有效性的判断依据。
  5. 记录所有成功的回环，并且在下次调用优化算法时读取回环检测的结果。

地图的导出

多数应用中，希望控制实时点云的载入规模，只加载自身周围200米范围内的点云，其他范围的点云视情况卸载，重新组织本章建立的点云，按照100米的边长进行分块，设计分块加载和卸载的接口。点云的切分实际上是根据每个点的坐标计算其所在的网格，然后将它投到对应的网格中。切分完地图之后，没快递图会独立到处一个PCD文件，程序可以快速读取到地图区块的位置。

激光雷达传感器的分类与数学模型

Lidar(Light detection and ranging)即激光雷达，与其他领域Radar(radio detection and ranging）表述的雷达有所不同，其中自动驾驶中还有一种毫米波雷达被称为Radar.

自动驾驶使用的激光雷达主要分为机械旋转式激光雷达和固态激光雷达两种

  1. 机械旋转式激光雷达（Spining Lidar）:可以看成是一列以固定频率旋转的激光探头。每个探头都能够快速探测外部物体离自身的距离。每旋转一周就可以完成一次对周围场景的扫描。可以根据线数进一步细分，常见包括4、8、16、32、64、80、128线等。
  2. 固态激光雷达（Solid State Lidar）：固态激光雷达本身不会进行360°扫描，只能探测约120°视野范围的3D信息，与RGBD相机十分相似。

扫描方式 整体宏观机械旋转 微观镜片偏转 (MEMS) 或 无机械运动 (OPA/Flash)

视场角 单个可实现360°水平覆盖 单个视场角有限（如120°），需多个拼接
可靠性/寿命 较低（有磨损，不耐振动） 很高（无/微运动部件，抗振性好）
成本 非常高 量产后有潜力变得很低
体积/重量 大、重 小、轻，易于集成
技术成熟度 非常成熟 发展中，MEMS相对成熟
主要应用 自动驾驶原型车、机器人、高精度地图测绘 大规模量产的乘用车、工业自动化、消费电子

最近邻问题

在实际应用中，不止关心最近的一个点，而是更关心以下两种扩展形式

  1. k-最近邻（K-Nearest Neighbors）：找到距离查询点最近的k个点，例如找到距离最近的5个点
  2. 半径最近邻（Radius Neighbors Search）:找到距离查询点某个半径r 范围内所有的特征点，如找到发哦周围100米 内所有邻居

一个点的特性必须通过分析它和周围邻近点的关系体现，因此，最近邻搜索几乎是所有点云处理算法的基石和第一步，有很多处理思路

暴力最近邻

暴力最近邻（Brute-force Nearest Neighbour Search）是最简单直观的最近邻搜索。遍历每一个点，找到最近的那一个，至于暴力K近邻则是找到最近的K个。

栅格与体素方法

栅格核心思想： 将一个2D平面分割成一个由许多个大小相同的小方块（单元格或像素） 组成的规则网格。每个小方块（Cell）都有一个唯一的坐标（如行号和列号），并被赋予一个值来表示该区域的某种属性。

体素核心思想：将一个3D空间分割成一个由许多个大小相同的小立方体（体素） 组成的规则网格。每个小立方体就是一个体素，它也有一个唯一的3D坐标（如X, Y, Z索引），并被赋予一个值来表示该3D空间区域的属性。

拟合问题

在传统SLAM应用中，关心的元素是基本的、静态的，更关心如何使用这些元素帮助我们进行点云之间的匹配和配准。在配准问题中，常见的做法是先利用最近邻结构找到一个点的若干最近邻，再对这些最近邻进行拟合。

平面拟合

简单来说，就是从一个三维空间中的点云集合中，找出一个“最理想”的平面，使得这个平面能够最好代表这些点的整体分布趋势，而最理想通常意味着所有点到这个平面的总距离之和最小。

解决平面拟合问题最主流的方法：

  1. 最小二乘：适用于当点云数据相对干净，大部分点都确实分布在一个平面附近，只有少量高斯噪声的情况下。这个问题可以使用的技术是主成分分析（Principal Component Analysis, PCA）

  1. RANSAC（Random Sample Consensus 随机采样一致性）：

在实际应用中，通常会将两者结合：

  1. 首先使用RANSAC从包含各种物体和噪声的原始点云中，稳健识别属于某个平台的内点集合

  2. 然后，对于只包含内点的集合应用PCA，精确计算出最终平面方程。

这样利用了RANSAC鲁棒性，又利用最小二乘的精确性。

直线拟合

与平面拟合一样，仍然可以使用RANSAC和最小二乘法来拟合直线，使用参数方程来描述直线，而用一个一般方程描述平面。

第七章：3D SLAM

三维激光雷达的信息比二维激光雷达的信息更丰富，不容易被遮挡，能够更好地重建场地的三维结构，既可以对三维点云直接进行匹配，也可以提取一些几何特征之后再配准。

多线激光雷达的工作原理

机械旋转激光雷达

多线激光雷达的工作原理可以概括为：

在垂直方向上同时发射多数不同角度的激光，通过整体的水平旋转，将这些线扫描成为面，最终利用ToF测距原理，实时绘制出周围环境的三维点云地图。

固态激光雷达

机械旋转式激光雷达必须让激光探头转动起来，才能探测周身360°范围内的距离，所以，它们不可避免地要内置一套精密的运动机构。所以在车辆震动等恶劣条件下容易损坏。所以就出现了固态激光雷达。

类型 扫描方式 运动部件 关键优势
传统机械式 马达带动整体旋转 有（宏观） 360°扫描，技术成熟
MEMS（半固态） 微振镜高速摆动 有（微观） 体积小，成本可控
OPA（真固态） 光学相控阵（相位干涉） 无 扫描极快，无磨损
Flash（真固态） 瞬间照亮整个场景 无 瞬时成像，无运动模糊

其核心就是通过一个高速旋转的马达，带动整个激光收发阵列进行360°扫描，这种设计包含宏观的、高速运动的机械部件。

多线激光雷达的扫描匹配

多线激光雷达的扫描匹配效果通常好于单线激光雷达，所以后端的处理更加简单，大多数多线激光雷达的SLAM系统不使用子地图的概念，直接管理点云本身，点云的配准主要以Scan to Scan 或者Scan to Map 为主。扫描匹配在多线激光雷达的背景下，也常常被称为点云配准。

点到点ICP

算法目标是找到一个旋转矩阵R和平移向量t，使得所有最近点之间的平方和距离最小

实现基础的ICP算法，可以使用优化的思路，也可以使用解析的方法。如果利用线性代数的求解，可以使用SVD方法，如果利用非线性优化的方式求解，则可以考虑以迭代的方式去寻找最优值。

虽然点到点的ICP是基础，但是当它在处理稀疏、有噪声的多线LIDAR数据时有严重缺陷：

  1. 最近点≠对应点：有可能一个点在下一帧真正的对应点没有被扫描到，这样错误的匹配可能会将点云错误拉向最近点，使得点云平移估计不准
  2. 对离群点较为敏感：SVD求解的是最小二乘问题，对于所有点的误差平方求和，如果离群点太多，会产生巨大的误差，导致整个结果被带偏
  3. 收敛性：容易陷入局部最小值，这个跟初始的齐次变换有很大关系，如果偏差过大，可能会找到一个错误的对齐方式，并且卡住

点到线、点到面ICP

点到面ICP与点到点的ICP思路基本一致，有两个小的差异点：

  1. 需要给最近邻点拟合平面，因此需要查找多个最近邻，然后进行局部平面拟合。可以使用第五章提到的方法
  2. 需要添加一些阈值检查来判定平面是否合理，然后用平面参数计算高斯-牛顿函数的雅可比矩阵

NDT方法

NDT (Normal Distributions Transform，正态分布变换) 是一种与ICP（迭代最近点）截然不同的点云配准（Scan Matching）算法。NDT不关注单个点，关心的是点云的局部统计特性和形状。利用这些统计信息进行匹配就可以了。一组点云中最基本的统计信息就是均值和方差。

NDT算法核心是：

  1. 建模：将目标点云转换为一个连续的、分段的概率密度函数（PDF）
  2. 配准：调整源点云的位姿，使得源点云上的点在目标点云的概率模型下，其总似然最大。

算法详解：

与所有配准算法一样，NDT同样存在依靠初始估计的问题，另外，NDT也依赖体素大小，所有以来体素大小的算法会存在体素离散化的问题。

直接法激光雷达里程计

有了配准方法，可以配准多个点云，然后将它们拼在一起，形成局部地图，最终构建一个激光雷达里程计（Lidar Odometry）模块。在3D SLAM中，点云之间可以很容易进行合并，也可以将过去一段时间内的点云组成一个局部地图，然后将当前帧和这个局部地图进行配准。这种不需要提取特征的激光雷达里程计被称为直接法激光雷达里程计。其核心原理是：对齐-迭代-优化。而特征法则是将里程计求解分为：提取-匹配-优化三个步骤。

根据使用的配准方法不同，可以写出两种实现激光雷达里程计的思路

这两种思路的核心都是：扫描-地图匹配（Scan-to-Map）即使用当前的激光扫描去和一个已经建立的历史地图进行匹配，从而计算当前位姿。

最核心的区别就是历史地图的表示方式和更新方式。

左图的思路的历史地图是由原始点云构成的。通常使用ICP作为配准算法。当然也可以使用NDT作为配准算法。

右图思路的历史地图是由概率分布构成，明确使用NDT算法

使用NDT构建激光雷达里程计

如果两种算法都是用NDT来作为配准算法，左图会一直对当前的扫描数据和局部地图进行NDT配准，每隔一段距离或者一定角度就取一个关键帧，再把最近的若干个关键帧拼成一个局部地图，作为NDT算法的配准目标。配准的时候用上两帧的相对运动作为运动模型，预测当前帧的位姿，然后将它作为NDT的位姿初值。局部地图的合并、地图对应的体素、K-d树构建是里程计模块中主要消耗算力的部分。

使用这个算法，每次增加关键帧，都需要重新建立局部地图，同时要重置所有NDT体素。可以把当前的扫描数据直接放到NDT的体素中，同时对体素增加一个时间队列处理，让它自动丢弃很久没用到的那些体素，这个方法称为增量式NDT。

增量式NDT里程计

实现增量式NDT需要解决两个问题：

  1. 如何维护不断增加的体素
  2. 每个体素内的高斯参数应该如何改变

首先解决第一个问题

应该将体素数量限制在一定的范围内，那些很久之前建立的体素应该定期删除，需要维护一个近期使用的缓存机制（Least Recently Used Cache, LRU）。设置一个队列模型，把最近更新的体素放到最前面。当整个队列超出预期容量，就删除最旧的那部分体素。

高斯分布的增量式更新

不存储点本身，只需要存储计算均值还有协方差所需的累加值。

这样，实现的逻辑比之前的里程计更简单，只需要取一些关键帧，不断维护，往NDT内部添加新关键帧，然后调用配准函数既可。局部地图被放到NDT内部维护，不必再去拼接关键帧点云了。

特征法激光雷达里程计

特征的提取

先提取特征再做配准的激光雷达里程计，称为特征法激光雷达里程计（间接法）。在特征法里程计中，LOAM系列，和后续的各种改进版本，是自动驾驶行业中常用的开源方案，也是后续很多LIO系统的基础。

实时SLAM对特征有以下几个要求：

  1. 特征能反映点云的特点。我们不必直接对整个点云提取特征，而可以对单个线条上的扫描数据提取特征
  2. 在提取特征之后，应该很容易对这些特征点进行几何配准，可以使用前面的ICP和NDT方法。
  3. 特征提取不应该占用太多CPU或者GPU资源
  4. 整个激光雷达里程计或SLAM系统必须使用同样的计算结构，不是一部分运行在CPU上，一部分运行在GPU上。

基于激光雷达线束的特征提取

多线激光雷达带有线束信息，以及额外的信息：

  1. 某个激光点来自哪一条扫描线
  2. 同一条线上的激光点的先后顺序关系
  3. 有的可以输出各点的极坐标角度、扫描时间等精确信息

根据这些点的先后顺序，可以计算曲率：沿着同一根线束的曲率，如果曲率较大，就认为是角点，否则就认为是平面点。这样可以一定限度地提取特征，有助于进一步优化配准结果。

特征提取部分的实现

特征提取算法主要步骤：

  1. 计算每个点的线束并且归类。
  2. 依次计算线束中每个点的曲率
  3. 把点云按照一周分为若干区间，选择其中曲率最大的若干点座位角点，其余作为平面点

特征法激光里程计的实现

为角点和平面点分别构建两个局部地图，两者的ICP会放在同一个优化问题中，从原理上来看，特征类算法对原始点云进行简单的特征分类，，再针对不同特征构建不同的优化问题。可以将地面、天花板分离出来单独处理，按照线束曲率的大小，把点云进一步分为角点、不尖锐的角点、平面点、不那么平的平面点。

在松耦合LIO系统中，依然是一个状态估计器来计算车辆自身的状态。IMU和轮速为这个估计器提供惯性和速度方面的观测源。点云匹配结果为这个系统提供位姿数据的观测源。估计器中的卡尔曼滤波部分和点云配准部分是相对解耦的，可以任意选择一个里程计位姿作为观测源。点云配准时，也可以使用状态估计器的预测输出，作为初始位姿估计进行配准。相对的，紧耦合LIO系统就是把点云的残差方程直接作为观测方程，写入观测模型，这种做法相当于在滤波器或者优化算法中内置一个ICP或者NDT，将各种配准方法以及惯导的递推方法结合，就得到了一种LIO方案。

自动驾驶与机器人中的SLAM技术——第三章：惯性导航与组合导航

IMU系统的运动学

激光SLAM

问题：

  1. 频率低
  2. 运动畸变
  3. 特征稀疏场景失效

惯性导航

IMU通常包含加速度计和陀螺仪，可以高频测量设备自身的三轴加速度和三轴角速度。

作用：

  1. 高频位姿估计
  2. 运动畸变校正
  3. 为激光匹配提供初始值

缺点：

IMU是通过对加速度和角速度进行积分来推算位姿的，这个过程存在累计误差。时间一长，哪怕是很小的噪声也会被不断放大，导致推算出的位置和姿态会像“脱缰的野马”一样飘走（即漂移），无法长期独立工作。

组合导航

组合导航就是将激光SLAM和惯性导航组合起来，形成一个激光惯性SLAM系统。核心思想：

用IMU高频、短时精确的特性，弥补激光雷达的低频和畸变问题；同时，用激光SLAM长期、全局精确的特性，来不断校正IMU的累计误差。

技术上主要分为两种融合方式：

  1. 松耦合：这是一种比较简单直接的融合方式。激光SLAM和IMU各自独立计算出自己的位姿，然后通过一个滤波器（如卡尔曼滤波器）将这两个“结果”进行融合。它实现简单，但因为没有利用原始数据，精度和鲁棒性相对较差。
  2. 紧耦合：这是一种更先进、更主流的融合方式。它将IMU的测量值（角速度、加速度）和激光雷达的原始特征（点、线、面）共同放入一个大的优化问题中进行求解（通常使用因子图优化或复杂的卡尔曼滤波器）。在这种框架下，激光的观测可以非常直接地校正IMU的状态（如零偏），IMU的数据也可以更精确地指导点云处理。紧耦合虽然复杂，但性能更优，鲁棒性更强，是当前研究和应用的热点。

这样就得到了纯粹的运动加速度，如果对速度进行一次时间积分，就得到了速度，对速度进行再次时间积分，就得到了位置。

需要注意的是，要把IMU放在车辆中心，否则可能会在车辆发生旋转和移动的时候，测量到由车辆旋转导致的离心力、科氏力和角加速度。会影响IMU的读数。

IMU测量方程中的噪声模型

IMU噪声由两部分组成：测量噪声和零偏。在实际卡尔曼滤波算法中，系统状态不仅包括姿态、位置、速度，还会把偏置作为一个状态量来进行在线估计和校正，抑制IMU的漂移。噪声是滤波器的重要输入，通常通过艾伦方差工具对IMU进行离线标定获得。

使用IMU进行航迹推算

在已知此时刻位置速度以及姿态的前提下，通过测量IMU数据，获得角速度以及加速度，来更新姿态以及位置速度。

航迹推算是一个迭代积分的过程，在每个时间步，先用陀螺仪积分更新姿态，再利用新姿态消除加速度计读数中的重力，最后对纯运动加速度进行二次积分来更新速度和位置。在实际系统中，必须与其他传感器（GPS、LiDAR、相机等）进行融合，由这些外部传感器来不断校正IMU的累积误差，从而实现长期、鲁棒的定位。

卫星导航

我们日常生活中最熟悉的“GPS”其实是美国开发的系统的名称，而这类系统的通用术语是GNSS（Global Navigation Satellite System，全球卫星导航系统）。卫星导航实际上可以提供车辆所需的所有定位信息，包括车辆的位置、姿态、速度等物理量。

卫星定位的最终精度受很多因素影响，人们提出各种校正技术消除已知误差。PPP（Precise Point Positioning 精密单点定位）、**RTK(Real-Time Kinematic 实时动态差分)**等技术取得进展，使得定位精度不断提升。

差分全球卫星导航系统中差分技术是消除误差最核心，最有效的方法之一，基本思想是“通过比较消除共同误差”

主要技术

  • SBAS (星基增强系统)：这是一种广域差分系统。它由多个地面站组成网络，计算出大范围（如整个大陆）的修正信息，然后通过地球同步卫星广播给用户。例如美国的 WAAS、欧洲的 EGNOS，以及覆盖日本的 MSAS。
  • RTK (实时动态) 技术：这是一种更高精度的载波相位差分技术。它不仅使用差分修正伪距，还利用了信号的载波相位进行测量。通过解算整周模糊度，RTK技术可以实现厘米级的实时定位精度，是测绘、精准农业和自动驾驶等领域的核心技术。

两个蘑菇头就是RTK接收器，通常使用两个接收器组成双天线方案，根据两根天线的位置差计算车辆实时朝向。

常见的世界坐标系

地球上最常见的坐标系就是经纬度坐标系，即地理坐标系。而UTM（Universsal Transverse Mercator Grid System）坐标系是将地球视为一个椭球体，投影至横躺圆柱体上，将其展开并且进行分区得到的。

使用UTM的优点是使用了米制单位，与其他传感器兼容性好，缺点是某些地区可能存在两个分区的跨界处，需要进行额外的坐标处理。

使用误差状态卡尔曼滤波器实现组合导航

RTK设备为我们提供一个不太稳定的位姿观测源，可以使用RTK与IMU结合，使用拓展卡尔曼滤波器形成组合导航算法。严格来说，是使用误差状态卡尔曼滤波器（Error-State Kalman Filter, ESKF）。

误差状态卡尔曼滤波器的核心思想是：不直接估计整个状态，而是将状态分为两部分：一个大的“标称状态”和一个小的“误差状态”。

  1. 标称状态 (x): 使用IMU的原始测量值，通过非线性运动学方程进行高频积分。这个状态包含我们主要关心的量：位置 p、速度 v、姿态 q（通常用四元数表示）。
  2. 误差状态 (δx): 使用一个线性的卡尔曼滤波器来估计标称状态中存在的微小误差。这个误差状态通常包括：位置误差 δp、速度误差 δv、姿态误差 δθ（一个小的旋转矢量）、以及IMU的偏置误差 ba,bg.

整个过程是一个经典的“预测-校正”循环。

速度观测量

在GINS（GNSS/INS）系统中，长时间缺少RTK数据，ESKF就变为纯靠IMU积分的递推模式。该模式下位移将快速发散， 因为缺少速度观测。大部分轮式机器人会携带一个编码器测定自身速度，进而推算自身的局部运动情况。这些速度观测与IMU结合，可以形成局部的航迹推算，可以简单将他们融入ESKF，作为速度观测量。

轮式编码器它被安装在车轮的转轴或电机上，用于测量车轮的旋转运动。通过测量轮子转了多少圈、转了多快以及朝哪个方向转，我们就可以间接计算出机器人或车辆的行驶距离、速度和方向。

其特点可以概括为：

  1. 单个轮式编码器只输出轮子转过的距离，可以对这个距离进行采样。如果每隔固定时间采样，就可以看成对速度的测量值。
  2. 如果本身在底盘上的角度是固定不动的，可以通过编码器计算车辆位移
  3. 轮子本身智能测量车辆在前进方向上的速度位移，无法测量平行方向上或者高度的速度位移
  4. 实际中，车轮容易打滑，轮子可能会发生空转，导致编码器虽然有输出，胆识车辆本身并没有前进的状态。

在代码中，可以添加其X方向上的速度作为观测模型，可以获得带有轮速的ESKF。
