源码：freespace_planner_node.cpp + freespace_planner_node.hpp

节点定位

  • 节点名：/planning/scenario_planning/parking/freespace_planner
  • 何时激活：上游 scenario_selector 切到 PARKING 场景时。
    • 切换条件 = 车当前位姿在 parking_lot 多边形内（scenario_selector_node.cpp:158 selectScenarioByPosition()，isInParkingLot 查的是 current_pose，第 162 行）
  • 节点职责：在 PARKING 场景下，把 “车位终点位姿 (goal)” 变成 “含倒车段的 Trajectory（velocity 可为负）”，供控制器跟随。
  • 算法可插拔：默认 astar（Hybrid A ），可选 rrtstar（RRT），由参数 planning_algorithm 决定。

💡 ROS1→ROS2：节点没有可见的 while(spin) 主循环，靠 “订阅/发布 + 定时器回调” 驱动。读 ROS2 节点用 五步法：构造 → 订阅 → 发布 → 定时器 → 核心调用。

关键类：FreespacePlannerNode

类声明：class FreespacePlannerNode : public rclcpp::Node（hpp L104-141）。

成员 类型 记得什么 / 作用
trajectory_pub_ 等 4 个 Publisher 往哪些话题 发 消息（见 4.2输出）
route_sub_ 等 4 个 Subscription 从哪些话题 收 消息（见 4.2输入）
timer_ Timer 定时启动（update_rate Hz）
tf_buffer_ / tf_listener_ TF 查坐标变换（costmap↔base_link）
algo_ unique_ptr<AbstractPlanningAlgorithm> ★ 算法句柄（可插拔点）：实际是 AstarSearch 或 RRTStar
current_pose_ / goal_pose_ PoseStamped 车现在哪 / 要去哪
trajectory_ / partial_trajectory_ Trajectory 规划出的整条轨迹 / 当前局部轨迹
reversing_indices_ vector<size_t> ★ 倒车切换点：轨迹里前进 ↔ 倒车的分界（侧方/倒车入库的关键）
route_ / occupancy_grid_ / scenario_ / odom_ 各消息 4 路输入的最新缓存
is_completed_ bool 泊车是否完成

两个重点成员：

  • algo_ 是接口类 AbstractPlanningAlgorithm 的指针，切换不同算法
  • reversing_indices_ 标出轨迹上 “速度符号翻转” 的点，是 倒车分段发布 的依据

① onTimer 主循环（L414-466）

② planTrajectory 调算法（L468-505）

节点层的工作只是：备好输入（costmap + 起终点）→ 调 makePlan → 把输出包成 Trajectory。

其余四个函数

  • isPlanRequired（L350-384）：trajectory_ 空 → 要规划；replan_when_obstacle_found 且前方轨迹有障碍 → 要规划；replan_when_course_out 且车偏离 > th_course_out_distance_m → 要规划；否则不重规划。
  • updateTargetIndex（L386-412）：到当前段终点（< th_arrived_distance_m）且车停稳（< th_stopped_velocity_mps）→ 取下一个 reversing_index 切段；若没有下一段 → is_completed_=true，发 is_completed。
  • initializePlanningAlgorithm（L530-552）：车辆外形外扩 vehicle_shape_margin_m；"astar" → AstarSearch（L545），"rrtstar" → RRTStar（L547）。
  • 构造函数（L221-287）：见 参数表 + 订阅/发布清单。
