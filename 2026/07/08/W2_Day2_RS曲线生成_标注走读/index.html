RS 曲线在规划框架中的定位与功能

汽车不能像机器人那样原地旋转，必须遵守非完整运动学约束，即受限于最小转弯半径。

  • RS功能：在最小转弯半径约束下、允许前进 + 倒车，连接两个位姿 (x, y, θ) 的最短路径**。是 Dubins 曲线推广。
  • 为什么泊车要用它：侧方/倒车入库必须倒车，普通几何曲线（只前进）做不到；RS 给出”前进段 + 倒车段”拼接的最短解。
  • 在这个模块的作用：RS 是 freespace_planning_algorithms 算法库里的几何工具，RS曲线并非直接求解从起点到终点的最终路径，而是作为图搜索算法（A* RRT*）的节点拓展工具。主要负责两个功能：
    1. 启发函数（heuristic）：估”从当前点到终点还多远”，引导 A* 往终点搜（astar_search.cpp:227）。
    2. shoot / 终结：搜到某点时用 RS 直奔终点，命中即接上**（Day3 详读）**。

在 Hybrid A* 中，RS 曲线常被用作 Shot Heuristic (解析解剪枝)：当算法探索到一个靠近终点的节点时，如果不考虑障碍物，直接用 RS 曲线连向终点。如果这条 RS 曲线在 Costmap 上不碰撞，搜索就直接宣告成功，极大地加速了计算。

关键类：ReedsSheppStateSpace（成员表）

类声明：class ReedsSheppStateSpace（hpp L93）。它不是 ROS 节点，是纯几何工具类（无订阅/发布/定时器）。读它看”持有什么状态 + 提供哪些几何方法”。

成员 类型 / 签名 行号 作用
rho_ double hpp L138 转弯半径（唯一状态），所有弧的半径
枚举 ReedsSheppPathSegmentType {RS_NOP, RS_LEFT, RS_STRAIGHT, RS_RIGHT} hpp L97 段形状（4 种，注意：方向不在这里）
reedsSheppPathType[18][5] static const 表 hpp L100 / cpp L623 18 种路径类型（每类是一组段形状序列）
内部类 ReedsSheppPath — hpp L103 一条路径的完整描述：type_ + length_[5] + totalLength_
struct StateXYT {x, y, yaw} hpp L121 输入位姿（位置 + 航向）
distance(s0, s1) 方法 hpp L130 / cpp L656 返回两位姿的RS 最短距离（米）
reedsShepp(s0, s1) 方法 hpp L133 / cpp L661 返回最短路径对象
interpolate(s0, path, seg) 方法 hpp L135 / cpp L670 沿路径采样一个点（给定弧长）
  • 方向（前进/倒车）不在枚举里，而在 length_[i] 的正负号：正=前进，负=倒车。枚举只编码”直/左/右”三种形状。
  • 路径类型是 18 类（不是网上常说的 48）。reedsSheppPathType[18][5]（cpp L623），48 是对称展开的理论数，代码用 18 规范行 + 对称变换实现。数组大小是[18,5]因为一个RS曲线最多由5个运动段组成。

关键函数

函数 行号 职责
reedsShepp(x,y,phi)（匿名命名空间） cpp L608 ★取最短入口：依次调 5 个分组生成器，全程保留最短路径
CSC / CCC / CCCC / CCSC / CCSCC cpp L158 / L235 / L332 / L430 / L575 5 个分组生成器：各算一类候选路径，用对称变换翻倍后留最短
distance(s0, s1) cpp L656 rho_ * path.length() —— 归一化长度 × 半径 = 实际米数
reedsShepp(s0, s1)（类方法） cpp L661 坐标转到起点系 + 除以 rho_ 归一化 → 调匿名 reedsShepp
interpolate(s0, path, seg) cpp L670 沿路径走 seg 弧长，逐段积分出目标点（见证曲率跳变）
8 个公式函数 LpSpLp 等 cpp L125–L555 各对应 RS 论文一个 formula（8.1–8.11），算单一几何解

**核心数学结算：**CSC等生成器，内部会调用LpSpLp等函数，然后再接着被核心函数reedsShepp()依次调用，每次调用都会比较当前算出的路径长度与已有最短长度比较，保留最短的一条。

**对外计算接口：**ReedsShepp,外界获取最短路径的方法，首相将起点与终点之间的相对差值，进行平移旋转，转化为相对于原点的目标点。除以最小转弯半径，把问题归一化到单位园的集合问题上，调用匿名函数reedsShepp结算最短路径。.

**轨迹离散化：*interpolate，会给定一条RS path 和一个长度 seg (通常是离散步长，如0.1米)，通过积分计算出在0.1米处，车辆实际坐标。这样，可以使得外层A等全局算法，每隔0.1米采样一个点，然后去Costmap上确认是否撞墙。

下面贴最关键的两个（reedsShepp 取最短 + CSC 分组生成器）带标注代码。

① reedsShepp 取最短入口（cpp L608）——整个 RS 生成的总调度

path 是引用传入每个分组函数，初值 length=max。每个分组里每算出一个候选 L，只要 L < path.length() 就覆盖 path。
所以”取最短”不是最后比一次，而是边算边贪心更新——5 个分组跑完，path 自然就是全局最短。

② CSC 分组生成器（cpp L158）——对称变换翻倍的样板

③ distance（cpp L656）——A* 启发函数最终调的就是它

这就是 Day1→Day2 的接口：A* 的 estimateCost(astar_search.cpp:227) → calcReedsSheppDistance(L31) → 本函数 → 给 A* 当”到终点还多远”的启发值。

RS 曲线生成逻辑

三种形状 × 两种方向 = 6 基元

RS 把任意路径切成”段”，每段是下面 3 种形状之一，方向由长度正负决定：

形状 枚举（hpp L97） 曲率 弧长正 弧长负
直行 RS_STRAIGHT 0 前进直行 倒车直行
左转弧 RS_LEFT +1/rho_ 前进左转 倒车左转
右转弧 RS_RIGHT −1/rho_ 前进右转 倒车右转
（占位） RS_NOP — 未用段填这个 —

→ 3 形状 × 2 方向 = 6 基元（S+/S−/L+/L−/R+/R−）。RS_NOP 仅用于把不足 5 段的路径补齐。

公式函数命名就是这个意思：LpSpLp = Left-plus、Straight-plus、Left-plus（全前进的”左-直-左”）；m=minus=倒车，如 LpRmL=左前-右倒-左前。

路径类型表：18 类（cpp L623–642）

每类是一组”段形状序列”（最多 5 段，多的位填 RS_NOP）：

组 类型行 段形状序列 举例含义
CCC 0 / 1 L-R-L / R-L-R 三圆弧（如 LRL）
CCCC 2 / 3 L-R-L-R / R-L-R-S/… 四圆弧
CCSC 4–11 含一段直线的 4 段组合 圆-圆-直-圆 等
CSC 12–15 L-S-R / R-S-L / L-S-L / R-S-R 圆-直-圆（最常见，如 LSL/LSR）
CCSCC 16 / 17 L-R-S-L-R / R-L-S-R-L 五段组合

生成 + 取最短流程

关键认知：分段常曲率，G1 连续但曲率不连续

看 interpolate（cpp L670）就懂——它逐段走，每段是恒定曲率的运动：

含义：

  • 每段内部曲率恒定（直=0，弧=±1/rho_）。
  • 段与段交界处曲率跳变（如 0 → 1/rho_，或 +1/rho_ → −1/rho_）。
  • 所以 RS 路径是 G1 连续（位置 + 切向连续，车能不”瞬移”地跟），但不是 C² 曲率连续。

若 W3/W4 验收口径要求”曲率连续“，RS 单独不满足（段间曲率阶跃）——需要后接样条/回旋曲线（clothoid）平滑，或由控制器做限幅。

关键参数

RS 工具类本身只有 1 个参数：rho_（转弯半径，hpp L138）。它不是节点参数，而是由 A* 传进来的：

参数 来源 Day1 默认值 对 RS 的影响
rho_（turning radius） A* 的 minimum_turning_radius 传入（astar_search.cpp:227 的 radius） 0.5 m 半径越小，车能转的弯越急，RS 弧段越短

minimum_turning_radius 是 Day1 参数表（cpp L295）里的算法参数，Day3 读 A* 时会看到它怎么传给 ReedsSheppStateSpace。Day2 只需知道：rho_ 决定 RS 所有弧的半径。
