第五章：基础点云处理

激光雷达传感器与点云的数学模型

激光雷达传感器的分类与数学模型

Lidar(Light detection and ranging)即激光雷达，与其他领域Radar(radio detection and ranging）表述的雷达有所不同，其中自动驾驶中还有一种毫米波雷达被称为Radar.

自动驾驶使用的激光雷达主要分为机械旋转式激光雷达和固态激光雷达两种

  1. 机械旋转式激光雷达（Spining Lidar）:可以看成是一列以固定频率旋转的激光探头。每个探头都能够快速探测外部物体离自身的距离。每旋转一周就可以完成一次对周围场景的扫描。可以根据线数进一步细分，常见包括4、8、16、32、64、80、128线等。
  2. 固态激光雷达（Solid State Lidar）：固态激光雷达本身不会进行360°扫描，只能探测约120°视野范围的3D信息，与RGBD相机十分相似。
特性 机械旋转式激光雷达 固态激光雷达
扫描方式 整体宏观机械旋转 微观镜片偏转 (MEMS) 或 无机械运动 (OPA/Flash)
核心部件 电机、旋转结构、多个激光收发单元 MEMS振镜、光学相控阵、面阵传感器等
视场角 单个可实现360°水平覆盖 单个视场角有限（如120°），需多个拼接
可靠性/寿命 较低（有磨损，不耐振动） 很高（无/微运动部件，抗振性好）
成本 非常高 量产后有潜力变得很低
体积/重量 大、重 小、轻，易于集成
技术成熟度 非常成熟 发展中，MEMS相对成熟
主要应用 自动驾驶原型车、机器人、高精度地图测绘 大规模量产的乘用车、工业自动化、消费电子

最近邻问题

在实际应用中，不止关心最近的一个点，而是更关心以下两种扩展形式

  1. k-最近邻（K-Nearest Neighbors）：找到距离查询点最近的k个点，例如找到距离最近的5个点
  2. 半径最近邻（Radius Neighbors Search）:找到距离查询点某个半径r 范围内所有的特征点，如找到发哦周围100米 内所有邻居

一个点的特性必须通过分析它和周围邻近点的关系体现，因此，最近邻搜索几乎是所有点云处理算法的基石和第一步，有很多处理思路

暴力最近邻

暴力最近邻（Brute-force Nearest Neighbour Search）是最简单直观的最近邻搜索。遍历每一个点，找到最近的那一个，至于暴力K近邻则是找到最近的K个。

栅格与体素方法

栅格核心思想： 将一个2D平面分割成一个由许多个大小相同的小方块（单元格或像素） 组成的规则网格。每个小方块（Cell）都有一个唯一的坐标（如行号和列号），并被赋予一个值来表示该区域的某种属性。

体素核心思想：将一个3D空间分割成一个由许多个大小相同的小立方体（体素） 组成的规则网格。每个小立方体就是一个体素，它也有一个唯一的3D坐标（如X, Y, Z索引），并被赋予一个值来表示该3D空间区域的属性。

拟合问题

在传统SLAM应用中，关心的元素是基本的、静态的，更关心如何使用这些元素帮助我们进行点云之间的匹配和配准。在配准问题中，常见的做法是先利用最近邻结构找到一个点的若干最近邻，再对这些最近邻进行拟合。

平面拟合

简单来说，就是从一个三维空间中的点云集合中，找出一个“最理想”的平面，使得这个平面能够最好代表这些点的整体分布趋势，而最理想通常意味着所有点到这个平面的总距离之和最小。

解决平面拟合问题最主流的方法：

  1. 最小二乘：适用于当点云数据相对干净，大部分点都确实分布在一个平面附近，只有少量高斯噪声的情况下。这个问题可以使用的技术是主成分分析（Principal Component Analysis, PCA）

  1. RANSAC（Random Sample Consensus 随机采样一致性）：

在实际应用中，通常会将两者结合：

  1. 首先使用RANSAC从包含各种物体和噪声的原始点云中，稳健识别属于某个平台的内点集合

  2. 然后，对于只包含内点的集合应用PCA，精确计算出最终平面方程。

这样利用了RANSAC鲁棒性，又利用最小二乘的精确性。

直线拟合

与平面拟合一样，仍然可以使用RANSAC和最小二乘法来拟合直线，使用参数方程来描述直线，而用一个一般方程描述平面。
