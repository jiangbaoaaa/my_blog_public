第九章：自动驾驶车辆的离线地图构建

发表于 2025-11-03 分类于 自动驾驶与机器人中的SLAM技术

第九章：自动驾驶车辆的离线地图构建

点云建图的流程

上图是整个建图流程：

  1. 从给定的ROS数据包中解出IMU、RTK和激光雷达数据。使用IMU与激光雷达数据组成LIO系统。
  2. 大部分自动驾驶车辆会携带RTK设备或者组合导航设备。按照RTK或者组合导航的位姿给每个关键帧赋一个RTK位姿，作为观测。
  3. 使用LIO作为相邻帧运动观测，使用RTK位姿作为绝对坐标观测，优化整条轨迹并且判定各关键帧的RTK有效性。这是第一轮优化。
  4. 在上一步的基础上对地图进行回环检测，可以使用简单的基于欧氏距离的检测，使用NDT或者常见的配准算法计算相对位姿。
  5. 再把信息放到统一的位姿图中进行优化，消除累计误差带来的重影，同时移除失效区域。这是第二轮优化。确定位姿之后，可以按照这些位姿到处点云地图。为了方便地图加载和查看，可以对点云进行切片处理。处理之后的点云可以用来进行高精度定位或者地图标注。

前端实现

  1. 将ROS数据包中的RTK数据提取出来，放在RTK消息队列中，按照采集时间排序
  2. 用第一个有效的RTK数据作为地图原点，用其他的RTK读书减去原点，作为RTK位置观测
  3. 用IMU和激光雷达输出数据运行LIO，按照距离和角度阈值抽取关键帧，再保存关键帧结果。抽取关键帧时，也会从LIO中获取它的位姿和点云

后续的建图流程可以从前端的结果出发，不必解析ROS数据包。先使用RTK位姿和LIO位姿进行联合优化，在此基础上进行回环检测，消除累计误差。

后端位姿图优化和异常值检验

我们希望最终优化的轨迹能够考虑到各种传感器的输入，因此后端图优化需要包含以下因子：

  1. RTK因子。这是一种绝对的位姿观测数据。
  2. 雷达里程计信息，使用前面介绍的激光雷达里程计或者LIO作为局部连续性约束
  3. 回环因子，执行回环检测就加载回环检测的相关信息，把它们放入图优化中。

回环检测

点云地图的质量很大程度上取决于回环检测是，设计回环检测的步骤如下：

  1. 遍历第一轮优化轨迹中的关键帧。对每两个在空间上相隔较近，但是在时间上存在一定距离的关键帧进行一次回环检测。
  2. 对于每个关键帧对，使用Scan to Map 的配准方法，在关键帧附近抽取一定数量的点云作为子地图，然后对扫描和子地图进行配准。
  3. 由于每个检查点的计算都是独立的，使用并发编程让第二步能够在机器上并发执行，加快速度
  4. 配准的实际执行由NDT完成，使用NDT的分值作为回环有效性的判断依据。
  5. 记录所有成功的回环，并且在下次调用优化算法时读取回环检测的结果。

地图的导出

多数应用中，希望控制实时点云的载入规模，只加载自身周围200米范围内的点云，其他范围的点云视情况卸载，重新组织本章建立的点云，按照100米的边长进行分块，设计分块加载和卸载的接口。点云的切分实际上是根据每个点的坐标计算其所在的网格，然后将它投到对应的网格中。切分完地图之后，没快递图会独立到处一个PCD文件，程序可以快速读取到地图区块的位置。
